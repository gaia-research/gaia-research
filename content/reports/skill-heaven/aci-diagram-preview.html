The "Why": Literature Justification

Tier 0: Off Mode

Mechanism: No tools exposed. Relies purely on parametric knowledge.

Why? The EMNLP 2024 paper "Context Length Alone Hurts LLM Performance" proves that simply inflating the context window degrades reasoning by up to 85%, even with perfect retrieval. Exposing tools during purely architectural/logical tasks wastes bandwidth and triggers the SMART "tool overuse bias."

Tier 1: Low Mode (Skill Heaven)

Mechanism: Exposes K=1 tool per step using strict precondition-effect causal graphs.

Why? Causal Minimal Tool Filtering (CMTF) and "Looking Is Not Picking" showed that dense semantic retrieval causes "ToolChoiceConfusion" (logit-level representational collisions). By enforcing strict causality, CMTF achieves 0.99 pass rates while cutting token usage by 90%.

Tier 2: Native Mode (Balanced)

Mechanism: A dynamic shortlist (K≈5-7) of bounded native primitives (search, edit).

Why? SWE-agent proved that a constrained interface (e.g., 100-line viewer, syntax-guarded edits) outperforms raw linux shells (18.0% vs 11.0% pass rate). CodeAct proved that a unified, minimal action space vastly improves LLM System 2 planning over heavily fragmented JSON schemas.

Tier 3: High Mode (Topological)

Mechanism: Lazy-loads detailed schemas (K≈16) by traversing a Directed Acyclic Graph (DAG).

Why? ToolNet demonstrated that flat lists fail at scale, but ~80% of tools have fewer than 6 logical successors. GraSP showed that compiling tools into DAGs allows the ACI to enforce execution order and isolate failures (O(depth)) without dumping massive schemas into the prompt.

Tier 4: Max Mode (Skill Hell)

Mechanism: Access to the entire registry via multi-agent delegation. Prompt exposure hard-capped at K≈64. Requires rigid firebreaks.

Why? AnyTool proved that dumping 16k APIs into a 128k GPT-4 window collapses performance to 14%, establishing 64 as a prompt saturation limit. To mitigate hallucination, Tool-MVR mandates a secondary verifier agent to reflect on tool syntax before execution (boosting correction to 58.9%). Isolated Planning prevents cross-tool prompt poisoning.
